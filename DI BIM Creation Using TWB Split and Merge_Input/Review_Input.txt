Reviewer Validation Checklist

Completeness Checks

- All mandatory BIM JSON root-level keys (name, compatibilityLevel, model) are present in the output.
- All tables, columns, and measures from the DDL (ExtractCol1 1.txt) are included in the BIM output with exact names, data types, and roles.
- All visuals, dashboards, and measures from the visual extraction (ExtractVis1 1.txt) are represented in the BIM output as required.
- The Master Calendar table is present in the tables array with all 27 ADDCOLUMNS columns, as per the strict template.
- The three mandatory Master Calendar relationships are present in the relationships array.
- The output includes all required annotations, including "__PBI_TimeIntelligenceEnabled" set to "0" and "PBI_QueryOrder" listing all tables including "Master Calendar".
- All required partitions for physical tables use "mode": "import" and have the correct M-query structure.
- All measures required by the visual extraction and DDL are present and placed inside the correct physical table.
- The output is pretty-printed with 2-space indentation and contains no markdown or explanations.
- The S3FileWriterTool is called exactly once with the complete, pretty-printed BIM JSON.
- All calculated columns and measures are included as specified in the input files and instructions.
- No required table, column, or measure from the DDL or visual extraction is omitted.
- The roles array is present and correctly populated based on RLS detection.
- All required cultures and model-level annotations are included.
- The output starts with "{" and ends with "}" and is valid JSON.

Accuracy Checks

- All table and column names in the BIM output match the DDL exactly, including spaces and casing.
- All data types in the BIM output match the DDL-to-BIM mapping rules.
- No lineageTag property is present in any table, column, or measure.
- No cardinality property is present in any relationship.
- The Master Calendar partition uses "source.type": "calculated" and the exact DAX expression as specified.
- No EDATE() function appears in any DAX expression; only approved integer arithmetic is used for date windows.
- COUNTD() is replaced with DISTINCTCOUNT() in all DAX measures.
- No measure name is identical to any column name in the same table.
- All DAX expressions use table and column names exactly as in the DDL.
- All relationships have only the five allowed properties (name, fromTable, fromColumn, toTable, toColumn).
- All M-query partition expressions are arrays of strings with commas between every let step.
- No hallucinated tables, columns, or measures are present; all content is traceable to the DDL or visual extraction.
- All calculated columns reference only other columns and never themselves.
- All measures required by the visual extraction are present and correctly mapped.
- The output contains no markdown, explanations, or extraneous text.
- The S3FileWriterTool content parameter receives the result of json.dumps(obj, indent=2).
- All required relationships, including those for Master Calendar, are present and correct.
- The output does not include a MeasureTable; all measures are inside physical tables.
- All formatting and structure rules from the instructions are followed exactly.
- The output is directly parseable by json.loads() with no syntax errors.

Efficiency Checks

- The S3FileWriterTool is called exactly once; no duplicate or repeated tool calls occur.
- No duplicate tables, columns, measures, or relationships are present in the output.
- No redundant explanations, markdown, or extraneous sections are included.
- Output is concise and contains only the required keys, tables, and properties as per the instructions and input files.
- No unnecessary or repeated gap entries, dependency mappings, or findings are present.
- Similar findings, measures, or columns are logically grouped and not repeated.
- Output structure is optimized for readability and review efficiency, with clean formatting and no excessive verbosity.
- No unnecessary sections, such as MeasureTable or extra calculated tables, are generated.
- All tables have only the required annotations and properties; no extra or redundant metadata is present.
- Output is token-efficient, with no repeated content from the input files unless required for completeness.
- Large sections (e.g., Master Calendar) are included only once and not duplicated.
- All required validations and checks are present without redundancy.
