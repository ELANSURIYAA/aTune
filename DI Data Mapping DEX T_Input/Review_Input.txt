Reviewer Validation Checklist

Completeness Checks

- Verify that the output includes all mandatory sections as specified in the agent instructions (overall, configuration, mappingResults, entityScores, aiInsights).
- Confirm that all required fields in the output JSON (e.g., matchScore, status, mappingRule, reasoning, dataQualityCheckRules) are present and populated for every mappingResults entry.
- Ensure that every target column from every target table (as would be defined in the input) is represented in mappingResults, with no omissions.
- Check that all required summary statistics (matchPercentage, totalAttributes, approvedCount, pendingCount, rejectedCount, autoApprovedCount) are included in the overall block.
- Validate that all configuration details (source/target schema, tables, businessContext, aiHints) are present and populated.
- Confirm that all required AI insights sections (implicitRelationships, alternateMates, domainGroupings) are included, even if empty.
- Ensure that every mappingResults entry has a corresponding matchTable row with matching values.
- Check that all required status values (APPROVED, AUTO_APPROVED, PENDING, REJECTED) are used appropriately where applicable.
- Verify that the output structure strictly follows the expected JSON template with no missing blocks.
- Confirm that all required domain tags are present for every table and mapping entry.

Accuracy Checks

- Confirm that the output format strictly matches the expected JSON schema, with correct field names and nesting.
- Ensure that no fabricated or hallucinated content is present in the output (e.g., no source-side content invented if not available from knowledge bases).
- Validate that all status assignments (APPROVED, AUTO_APPROVED, PENDING, REJECTED) are justified according to the rules in the instructions.
- Check that all matchScore values are integers between 0 and 100 and align with the matching logic described in the instructions.
- Ensure that every mappingResults entry includes a non-empty domain and dataQualityCheckRules value.
- Confirm that no unsupported assumptions or content are introduced in the output.
- Verify that all transformation rules and reasoning fields are populated and align with the mapping logic.
- Ensure that the output contains no contradictions or inconsistencies across sections.
- Validate that all terminology and field values are used consistently and as defined in the instructions.
- Confirm that the output contains no extraneous or missing fields beyond those specified in the template.

Efficiency Checks

- Ensure that no duplicate mappingResults or matchTable entries are present.
- Confirm that redundant explanations, repeated gap entries, or unnecessary sections are avoided.
- Check that the output is concise, well-structured, and free of excessive verbosity.
- Validate that similar findings are logically grouped where appropriate and that overlapping findings are consolidated.
- Ensure that tables, statistics, and insights are not unnecessarily repeated in the output.
- Confirm that formatting is clean, optimized, and improves readability for reviewers.
- Ensure that the output is token-efficient while maintaining completeness and accuracy.
- Check that duplicate validations or checks are eliminated from the output.
- Validate that the output structure supports efficient review and traceability of all required elements.
