Reviewer Validation Checklist

Completeness Checks

- Verify that the output contains all mandatory sections as specified in the agent instructions, including overall summary, configuration, mappingResults, entityScores, and AI insights.
- Confirm that all required fields in the expected JSON output (such as matchScore, status, mappingRule, reasoning, domain, dataQualityCheckRules, etc.) are present and populated.
- Ensure that every section and field described in the Expected Output and Agent Description is included in the output, even if no input files were provided.
- Check that the output structure strictly follows the JSON template provided in the instructions.
- Validate that the output does not omit any required block or field, even in the absence of input files.
- Confirm that the output includes appropriate handling or messaging for missing input content, as per the agent’s error-handling instructions.

Accuracy Checks

- Ensure that the output aligns with the agent instructions and expected output format, with no fabricated or hallucinated content.
- Confirm that all fields and sections are populated with values that are justified by the available instructions and metadata.
- Check that no unsupported assumptions or content are introduced in the absence of input files.
- Verify that error or missing input handling in the output matches the rules specified in the agent instructions.
- Confirm that terminology and field names are used exactly as specified in the instructions and expected output template.
- Ensure that the output does not contradict any requirements or constraints from the agent instructions.

Efficiency Checks

- Confirm that the output does not include redundant sections or repeated explanations.
- Ensure that the output remains concise and does not introduce unnecessary verbosity in the absence of input files.
- Check that the output structure is clean, optimized, and easy to review.
- Verify that no duplicate fields or sections are generated in the output.
- Ensure that the output is token-efficient and avoids unnecessary repetition of content or structure.
