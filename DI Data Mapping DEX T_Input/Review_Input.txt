Reviewer Validation Checklist

Completeness Checks

- Verify that all mandatory sections specified in the agent instructions are present in the output JSON.
- Confirm that all required fields in the expected output template are included and populated.
- Ensure that all required analyses (mappingResults, entityScores, aiInsights, configuration, overall) are covered.
- Check that all dependencies and referenced files mentioned in the instructions are addressed or marked as unavailable if not present.
- Validate that all formatting and structural requirements from the instructions are followed.
- Confirm that no required section or field from the expected output is skipped.
- Ensure that all gap/missing item handling requirements are covered as per instructions.

Accuracy Checks

- Confirm that the output strictly aligns with the provided agent instructions and expected output template.
- Ensure that no hallucinated content or unsupported assumptions are introduced in the output.
- Validate that all status, matchScore, and rule assignments follow the logic and thresholds defined in the instructions.
- Check that all findings and mappings are traceable to the instructions or input (if available).
- Ensure that terminology and field names are used consistently and as specified in the instructions.
- Confirm that the output structure and field population match the required JSON schema exactly.
- Verify that no instruction is misinterpreted or partially implemented.

Efficiency Checks

- Ensure that duplicate analysis, redundant explanations, and repeated gap entries are avoided.
- Confirm that the output remains concise and structured, without unnecessary sections or verbosity.
- Validate that formatting is clean and optimized for reviewer readability.
- Check that output is token-efficient while maintaining completeness.
- Ensure that similar findings are logically grouped where appropriate and redundant validations are eliminated.
