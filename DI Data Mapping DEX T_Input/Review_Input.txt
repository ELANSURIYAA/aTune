Reviewer Validation Checklist

Completeness Checks
- Verify that all mandatory sections specified in the agent instructions are present in the output JSON.
- Confirm that all required fields and blocks (overall, configuration, mappingResults, entityScores, aiInsights) are included in the output.
- Ensure that all required analyses and mapping logic described in the instructions are covered.
- Check that all referenced files/content (if any) are analyzed where required, or that their absence is correctly handled.
- Confirm that all formatting and structural requirements from the instructions are followed.
- Ensure that no requested section is skipped, even if no input file was found.
- Verify that the output includes appropriate handling for missing input files as per agent instructions.

Accuracy Checks
- Confirm that the output aligns with the provided agent instructions and expected output format.
- Ensure that no hallucinated content or unsupported assumptions are introduced in the absence of input files.
- Verify that all fields and sections are populated as per the instructions, or that missing input is explicitly handled.
- Check that terminology and field names are used consistently and as specified in the instructions.
- Ensure that the output does not contradict any requirements or constraints from the instructions.
- Confirm that the output does not fabricate data or mappings in the absence of input files.

Efficiency Checks
- Ensure that the output avoids redundant or unnecessary sections in the absence of input files.
- Confirm that the output remains concise and structured, with no excessive verbosity.
- Verify that no duplicate or repeated content is generated.
- Ensure that the output structure improves readability and review efficiency, even when input files are missing.
- Confirm that the output is token-efficient and does not include unnecessary placeholders or filler content.
