Reviewer Validation Checklist

Completeness Checks

- Verify that the output includes all mandatory sections as specified in the agent instructions (overall, configuration, mappingResults, matchTable, entityScores, aiInsights).
- Confirm that all required fields in the output JSON structure are present and populated (e.g., matchScore, status, mappingRule, reasoning, domain, dataQualityCheckRules, etc.).
- Check that every target column from every target table in the input (if provided) is represented in mappingResults, with no omissions.
- Ensure that all required analyses (implicitRelationships, alternateMates, domainGroupings) are included in aiInsights.
- Validate that all referenced files/content (e.g., glossary, profile report, DQ report) are analyzed if enabled in the input.
- Confirm that all required statistics (matchPercentage, totalAttributes, approvedCount, pendingCount, rejectedCount, autoApprovedCount) are included in the overall block.
- Ensure that all required configuration details (source/target schema, tables, businessContext, aiHints) are present.
- Check that all required mapping rules and data quality check rules are generated for each mappingResults entry.
- Confirm that all formatting and structural requirements for the JSON output are followed.
- If no input file is present, ensure the output handles this scenario gracefully and does not fabricate mappings.

Accuracy Checks

- Confirm that the output strictly aligns with the agent instructions and expected output format.
- Verify that no content is hallucinated or fabricated beyond what is supported by the instructions and input files.
- Ensure that all field values (e.g., matchScore, status, mappingRule, reasoning) are logically consistent and justified based on the instructions and input.
- Check that all dependency mappings, technical mappings, and domain tags are valid and instruction-compliant.
- Validate that all status assignments (APPROVED, AUTO_APPROVED, PENDING, REJECTED) follow the rules specified in the instructions.
- Confirm that all business rule interpretations and transformation rules are correct and traceable to the input or instructions.
- Ensure that all terminology and field names are used consistently and as defined in the instructions.
- Check that no unsupported assumptions or contradictions are introduced in the output.
- Validate that all required formatting standards (e.g., integer ranges, non-empty fields) are followed.
- If no input file is present, confirm that the output does not misinterpret the absence of input or generate false mappings.

Efficiency Checks

- Ensure that duplicate analysis, redundant explanations, and repeated gap entries are avoided in the output.
- Confirm that the output is concise, well-structured, and free from unnecessary sections or verbosity.
- Check that duplicate dependency mappings, risks, recommendations, and validations are eliminated.
- Validate that tables and findings are not unnecessarily repeated and that similar findings are logically grouped where appropriate.
- Ensure that the output structure improves readability and review efficiency.
- Confirm that the output is token-efficient while maintaining completeness and accuracy.
- If no input file is present, ensure that the output does not include unnecessary placeholders or boilerplate for missing content.
