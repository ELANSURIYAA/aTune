Reviewer Validation Checklist

Completeness Checks

- Verify that all mandatory sections specified in the agent instructions are present in the output.
- Confirm that all required fields and JSON blocks (overall, configuration, mappingResults, entityScores, aiInsights) are included as per the expected output template.
- Ensure that all required analyses and mappings described in the instructions are covered, even if no input file was provided.
- Check that all referenced metadata (agent role, goal, backstory, description, expected output) is reflected in the output structure and logic.
- Validate that the output includes all required status and classification fields (e.g., APPROVED, AUTO_APPROVED, PENDING, REJECTED) as described in the instructions.
- Confirm that all gap/missing item handling requirements (e.g., handling of missing input files) are addressed as per the instructions.
- Ensure that all formatting and structural requirements from the instructions are followed in the output.

Accuracy Checks

- Confirm that the output strictly aligns with the provided agent instructions and expected output template.
- Verify that no hallucinated content or unsupported assumptions are introduced in the absence of input files.
- Ensure that all status assignments and field values are justified according to the instructions.
- Check that all values, fields, and classifications match the requirements and logic described in the instructions.
- Validate that the output does not contradict any section of the agent instructions or expected output.
- Confirm that all error handling and gap reporting is accurate and instruction-aligned for missing input files.
- Ensure that terminology and field names are used consistently with the instructions and expected output template.

Efficiency Checks

- Verify that the output avoids unnecessary or redundant sections in the absence of input files.
- Confirm that the output remains concise and does not include excessive verbosity or repeated explanations.
- Ensure that no duplicate validations, mappings, or classifications are generated.
- Check that the output structure is clean, optimized, and reviewer-friendly.
- Validate that the output is token-efficient while maintaining completeness and instruction compliance.
- Confirm that large sections are not generated unnecessarily when input files are missing.
