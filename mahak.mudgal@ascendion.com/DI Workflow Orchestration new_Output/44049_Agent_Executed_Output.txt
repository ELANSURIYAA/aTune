Business Rules & Compliance Agent must re-validate that every business rule is linked to at least one entity or attribute, all PII classifications are assigned, and all audit trail requirements are documented before approval. | Criteria                       | Description               |

Entity & Attribute Modeling Agent: Use GPT-4o for ambiguity resolution tasks (resolving unclear entity definitions, inferring missing relationships, validating entity completeness) and route deterministic tasks (populating entity templates, assigning data types, generating ER diagrams) to Gemini Pro, reducing cost by ~35% for template-based subtasks. Business Rules & Compliance Agent: Use Claude Sonnet 3.5 for rule extraction and compliance validation tasks (extracting business rules, validating PII classifications, cross-referencing compliance standards) due to strong structured extraction performance at lower cost than GPT-4o, and route simple checklist validation tasks (verifying retention policies, logging audit requirements) to Gemini Pro, reducing cost by ~50% for high-volume validation subtasks. Model Synthesis & Integration Agent: Use GPT-4o for complex integration tasks (reconciling entity definitions, validating governance alignment, generating validation reports) and route deterministic assembly tasks (merging JSON schemas, generating handoff documentation, uploading artifacts) to Gemini Pro, reducing cost by ~30% for assembly subtasks. |
| Token size reduction           | Requirements Parsing & Validation Agent: Trim the input requirements document by removing boilerplate sections (cover pages, table of contents, appendices with non-functional content), summarizing narrative sections into bullet points, and caching the parsed requirements structure for reuse by downstream agents, reducing input token count by ~30% without losing critical content. Entity & Attribute Modeling Agent: Trim the input by receiving only the parsed requirements JSON (not the full original document), caching entity definitions as they are created to avoid re-processing, and summarizing ambiguity logs into concise escalation messages, reducing input token count by ~40% while maintaining full modeling context. Business Rules & Compliance Agent: Trim the input by receiving only the relevant sections of the requirements document (business rules, compliance, audit sections) rather than the full document, caching compliance standards and checklists for reuse, and summarizing rule extraction results into structured JSON, reducing input token count by ~35% without losing compliance coverage. Model Synthesis & Integration Agent: Trim the input by receiving only the final entity definitions, business rules, and compliance artifacts (not the full requirements document or intermediate outputs), caching the governance validation schema for reuse, and summarizing validation results into concise reports, reducing input token count by ~45% while maintaining full integration context. |
| Hybrid model (agent + non-agentic) | Requirements Parsing & Validation Agent: Use regex-based extraction for deterministic sections (extracting entity names from bulleted lists, extracting attribute data types from tables, extracting compliance checklist items) and reserve LLM calls for ambiguous or narrative sections (extracting business goals from paragraphs, inferring entity relationships from process descriptions), reducing LLM calls by ~25% for deterministic extraction tasks. Entity & Attribute Modeling Agent: Use rule-based logic for deterministic tasks (assigning default data types based on attribute names, validating primary key presence, checking relationship cardinality syntax) and reserve LLM calls for ambiguity resolution and inference tasks (resolving unclear entity definitions, inferring missing relationships), reducing LLM calls by ~30% for validation and template population tasks. Business Rules & Compliance Agent: Use rule-based logic for deterministic tasks (validating PII classification completeness, checking retention policy syntax, verifying audit trail field presence) and reserve LLM calls for rule extraction and compliance validation tasks (extracting complex calculation rules, cross-referencing compliance standards), reducing LLM calls by ~40% for checklist validation tasks. Model Synthesis & Integration Agent: Use rule-based logic for deterministic tasks (merging JSON schemas, validating schema completeness, generating ER diagram files) and reserve LLM calls for complex integration and validation tasks (reconciling conflicting entity definitions, validating governance alignment, generating validation reports), reducing LLM calls by ~35% for assembly and file generation tasks. |
