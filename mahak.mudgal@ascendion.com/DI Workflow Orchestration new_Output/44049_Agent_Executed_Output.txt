SECTION 4 — IMPROVEMENTS & OPTIMISATIONS

─────────────────────────────────────────────────────────────
ACCURACY — How well the agent produces correct, on-target outputs and avoids errors.

| Criteria            | Description                          |
|---------------------|--------------------------------------|
| Self-validation     | Requirements Parsing & Validation Agent must re-validate that all extracted requirements map back to specific sections in the source document and that no sections are skipped or duplicated. Entity & Attribute Modeling Agent must cross-check that every entity and attribute is traceable to a parsed requirement and that all relationships are bidirectionally consistent (e.g., if Application → Customer exists, Customer must reference Application). Business Rules & Compliance Agent must verify that every extracted rule has a corresponding compliance reference and that no conflicting rules exist. Model Synthesis & Integration Agent must validate that the final model covers 100% of requirements, all entities have complete attribute sets, and all relationships are documented before handoff. |
| Boundary definition | Requirements Parsing & Validation Agent must NOT alter, paraphrase, or invent requirements; must NOT proceed with modeling if critical sections are missing; must NOT resolve ambiguities without stakeholder input. Entity & Attribute Modeling Agent must NOT create entities or attributes that are not supported by parsed requirements; must NOT assume relationships without evidence; must NOT proceed if ambiguity exists. Business Rules & Compliance Agent must NOT omit compliance requirements; must NOT invent business rules; must NOT approve incomplete compliance documentation. Model Synthesis & Integration Agent must NOT hand off incomplete models; must NOT bypass governance validation; must NOT proceed if integration with governance systems fails. |
| Known issues log    | Requirements Parsing & Validation Agent must log all instances of missing sections, ambiguous language, conflicting requirements, and unresolvable gaps to build a pattern library of common requirement defects. Entity & Attribute Modeling Agent must log all entity ambiguities, missing attributes, unclear relationships, and escalation outcomes to refine entity identification patterns over time. Business Rules & Compliance Agent must log all conflicting rules, missing compliance references, and escalation resolutions to improve rule extraction accuracy. Model Synthesis & Integration Agent must log all integration failures, validation errors, and governance misalignments to prevent recurrence in future modeling cycles. |

─────────────────────────────────────────────────────────────
EFFICIENCY — How well the agent produces output that is most optimised — same correctness in fewer steps or words.

| Criteria                  | Description                    |
|---------------------------|--------------------------------|
| Best practices (sample outputs) | Requirements Parsing & Validation Agent should be provided with labelled examples of correctly parsed requirements (showing section headers, extracted data elements, and validation flags) and counter-examples of incomplete or incorrectly parsed documents (showing missed sections or invented content) to learn the expected extraction format and depth. Entity & Attribute Modeling Agent should receive sample entity definitions with complete attribute lists, relationship diagrams, and ambiguity logs, plus counter-examples showing incomplete or invented entities, to understand the required level of detail and documentation. Business Rules & Compliance Agent should be given examples of complete business rule catalogs with compliance references and audit trail specifications, plus counter-examples showing missing or conflicting rules, to learn the expected structure and completeness. Model Synthesis & Integration Agent should receive examples of fully validated and integrated conceptual models with handoff packages, plus counter-examples showing incomplete or unvalidated models, to understand the final deliverable format. |
| Defined output structure  | Requirements Parsing & Validation Agent must produce output in a fixed JSON schema with fields: {section_name, requirement_text, validation_status, gaps_identified, escalation_flag}. Entity & Attribute Modeling Agent must produce output in a structured format: {entity_name, attributes: [{name, type, constraints}], relationships: [{target_entity, cardinality, type}], ambiguity_log: [{issue, resolution}]}. Business Rules & Compliance Agent must produce output in a catalog format: {rule_id, rule_text, compliance_reference, policy_type, audit_requirement}. Model Synthesis & Integration Agent must produce output as a handoff package: {conceptual_model_diagram, entity_catalog, validation_report, governance_integration_log, audit_trail}. These fixed structures prevent verbose or wandering responses and ensure downstream agents can parse outputs reliably. |
| Define task breakdown     | Requirements Parsing & Validation Agent should decompose its work into: (1) Document structure extraction, (2) Section-by-section requirement parsing, (3) Gap identification, (4) Validation against checklist, (5) Escalation log generation. Entity & Attribute Modeling Agent should decompose into: (1) Entity candidate identification, (2) Attribute mapping per entity, (3) Relationship definition, (4) Ambiguity detection and escalation, (5) Model completeness validation. Business Rules & Compliance Agent should decompose into: (1) Business rule extraction, (2) Compliance reference mapping, (3) Security and retention policy documentation, (4) Audit trail specification, (5) Validation and escalation. Model Synthesis & Integration Agent should decompose into: (1) Model component assembly, (2) Completeness validation, (3) Governance system integration, (4) Handoff package preparation, (5) Final audit log generation. This task breakdown constrains reasoning overhead and enables efficient, step-by-step execution. |

COST EFFECTIVE — How to minimise cost per agent without sacrificing quality.

| Criteria                       | Description               |
|--------------------------------|---------------------------|
| Low cost model routing         | Requirements Parsing & Validation Agent: Use GPT-4o for initial document parsing and ambiguity detection (high complexity), then route validation checklist execution and gap logging to GPT-4o-mini (low complexity, rule-based). Entity & Attribute Modeling Agent: Use GPT-4o for entity identification and relationship inference (high complexity), then route attribute type assignment and completeness checks to GPT-4o-mini (low complexity, structured). Business Rules & Compliance Agent: Use Claude Sonnet 3.5 for rule extraction and compliance mapping (moderate complexity, structured), then route audit trail formatting and catalog generation to GPT-4o-mini (low complexity, templated). Model Synthesis & Integration Agent: Use Claude Sonnet 3.5 for model assembly and governance validation (moderate complexity, integration logic), then route handoff package formatting and audit log generation to GPT-4o-mini (low complexity, structured output). |
| Token size reduction           | Requirements Parsing & Validation Agent: Summarize or cache the requirements document structure (table of contents, section headers) after first parse to avoid re-processing the full document in subsequent validation steps; remove boilerplate text and focus only on data requirements sections. Entity & Attribute Modeling Agent: Cache the parsed requirements and reuse them across entity identification, attribute mapping, and relationship definition steps; remove redundant requirement text once entities are extracted. Business Rules & Compliance Agent: Extract and cache only the business rules and compliance sections from the requirements document; remove narrative or background sections that do not contain actionable rules. Model Synthesis & Integration Agent: Cache the entity catalog, attribute definitions, and relationship diagram after assembly; reuse them for validation and governance integration steps without re-generating the full model each time. |
| Hybrid model (agent + non-agentic) | Requirements Parsing & Validation Agent: Use regex or rule-based parsers to extract section headers, table structures, and numbered lists from the requirements document (deterministic, no LLM needed); route only ambiguous or unstructured text to the LLM for interpretation. Entity & Attribute Modeling Agent: Use a lookup table or database query to validate entity names and attribute types against enterprise standards (deterministic, no LLM needed); route only ambiguous entity definitions or unclear relationships to the LLM for reasoning. Business Rules & Compliance Agent: Use a compliance checklist or rule-based validator to cross-reference extracted rules against known regulatory standards (deterministic, no LLM needed); route only conflicting or missing rules to the LLM for escalation logic. Model Synthesis & Integration Agent: Use API calls to the governance system to validate entity names and attribute definitions (deterministic, no LLM needed); route only integration failures or validation errors to the LLM for resolution reasoning. |
