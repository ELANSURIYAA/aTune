Reviewer Validation Checklist

Completeness Checks

- Verify that all mandatory diagnosis fields specified in the expected output ({ticket_id, error_category, failing_layer, extracted_signals[], root_cause_in_different_module, root_cause_hint, notes}) are present in the output.
- Confirm that the output is structured as a JSON array of diagnosis objects as required.
- Ensure that all classification requirements (error_category, root_cause_in_different_module flag) are addressed as per the agent instructions.
- Check that all required extraction and flagging logic described in the agent description is implemented, even if no input file was provided.
- Validate that the output does not omit any mandatory sections or fields specified in the instructions.

Accuracy Checks

- Confirm that the output format strictly matches the expected JSON structure described in the instructions.
- Ensure that no hallucinated or unsupported fields are present in the output.
- Verify that all field names and values conform to the terminology and structure specified in the agent instructions.
- Check that no unsupported assumptions or extraneous content are introduced in the absence of input data.
- Ensure that the output does not contradict any explicit requirements from the agent instructions or expected output.

Efficiency Checks

- Confirm that the output does not contain redundant or duplicate diagnosis objects or fields.
- Ensure that the output remains concise and does not include unnecessary sections or explanations.
- Validate that the output is token-efficient and only includes the mandatory structure and fields as required by the instructions.
- Check that the output structure is optimized for reviewer readability and does not introduce inefficiency in the absence of input data.
