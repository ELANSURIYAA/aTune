Reviewer Validation Checklist

Completeness Checks

- Verify that the output contains a diagnosis object for each ticket if any input tickets are provided.
- Confirm that all mandatory fields are present in each diagnosis object: ticket_id, error_category, failing_layer, extracted_signals, root_cause_in_different_module, root_cause_hint, and notes.
- Ensure that all required error categories (as per instructions) are considered if relevant to the input.
- Check that the flag for root_cause_in_different_module is explicitly set in each diagnosis object.
- Validate that the expected JSON array structure is used for the output.
- Confirm that all formatting requirements specified in the instructions are followed.
- Ensure that if no input tickets are present, the output appropriately reflects this (e.g., empty array or explanatory note).

Accuracy Checks

- Confirm that the output format strictly matches the expected JSON array of diagnosis objects as described in the instructions.
- Ensure that no hallucinated or unsupported fields are present in the output.
- Verify that all mandatory fields are populated with logically valid values (e.g., boolean for root_cause_in_different_module).
- Check that the output does not introduce unsupported assumptions or content not justified by the input or instructions.
- Ensure that terminology and field names are used exactly as specified in the instructions.
- Confirm that the output does not contradict the agent’s instructions or expected output structure.
- Validate that the output does not misinterpret or partially implement any instruction requirements.

Efficiency Checks

- Ensure that duplicate diagnosis objects are not generated for the same ticket.
- Confirm that redundant explanations or repeated content are avoided in the output.
- Check that the output is concise and contains only the required fields and information.
- Ensure that the output structure is clean, readable, and optimized for review.
- Validate that unnecessary sections or fields are not included in the output.
- Confirm that the output is token-efficient and avoids excessive verbosity.
