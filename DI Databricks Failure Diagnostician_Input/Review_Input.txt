Reviewer Validation Checklist

Completeness Checks

- Verify that the output contains a diagnosis object for each ticket as required by the instructions.
- Confirm that all mandatory fields specified in the expected output ({ticket_id, error_category, failing_layer, extracted_signals[], root_cause_in_different_module, root_cause_hint, notes}) are present in each diagnosis object.
- Ensure that all required error categories mentioned in the instructions are supported and classified where applicable.
- Check that the output format is a JSON array as specified in the expected output.
- Validate that all mandatory flags (such as root_cause_in_different_module) are explicitly set as required.
- Confirm that all formatting requirements from the instructions are followed.

Accuracy Checks

- Ensure that the output aligns strictly with the provided instructions and expected output structure.
- Verify that no hallucinated or unsupported fields are present in the output.
- Confirm that all mandatory fields are populated with valid values (not left blank or null unless justified by the instructions).
- Check that the terminology and field names match exactly as specified in the instructions.
- Ensure that no unsupported assumptions or extra content are introduced.
- Validate that the output does not contradict any requirements from the instructions.

Efficiency Checks

- Confirm that duplicate diagnosis objects are not generated for the same ticket.
- Ensure that redundant fields or repeated explanations are avoided in the output.
- Check that the output remains concise and does not include unnecessary sections or excessive verbosity.
- Validate that the output structure is clean, optimized, and reviewer-friendly.
- Ensure that only the required fields and sections are present, with no superfluous content.
