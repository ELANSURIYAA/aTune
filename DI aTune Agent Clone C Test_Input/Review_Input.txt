Reviewer Validation Checklist

Completeness Checks

- Verify that all mandatory sections from the agent instructions are present in the output.
- Confirm that all required fields and analyses described in the instructions are addressed.
- Ensure that all referenced files and dependencies from the instructions are included or noted.
- Check that all required validation, logging, and error handling requirements from the instructions are covered.
- Validate that all formatting and output structure requirements from the instructions are followed.
- Confirm that the output includes all required summary and outcome sections as specified in the expected output.
- Ensure that the absence of input files is explicitly handled and documented if required by the instructions.

Accuracy Checks

- Verify that the output aligns with the provided agent instructions and expected output.
- Confirm that no unsupported assumptions or hallucinated content are introduced due to missing input files.
- Ensure that all references to input files are accurate and reflect their actual presence or absence.
- Check that all log entries, file paths, and URLs are constructed according to the instructions.
- Validate that all error handling and validation steps are accurately represented in the output.
- Ensure that terminology and formatting are consistent with the instructions.
- Confirm that the output does not contradict any requirements or rules specified in the instructions.

Efficiency Checks

- Verify that the output avoids redundant sections or explanations.
- Ensure that no unnecessary or duplicate validation checks are generated.
- Confirm that the output remains concise and does not include superfluous content due to the absence of input files.
- Check that the output structure is optimized for reviewer readability and efficiency.
- Ensure that all findings and checks are atomic and not combined or repeated.
- Validate that the output does not include unnecessary placeholders or repeated references to missing files.
- Confirm that the checklist is token-efficient and logically grouped for review.
