## Notes

### Instruction Sections Analysis:
- **Current Instruction Structure**: The instruction prompt contains only three fields: Agent Name, Description (empty), and Expected Output (empty)
- **Gap**: The instruction prompt is severely underdeveloped and lacks detailed specifications for the documentation generation task

### Mapping Observations:
1. **All gaps map to Expected Output section**: Since the Description and Expected Output fields are empty in the current instruction, all remediation actions are additions to the Expected Output section
2. **No existing content to delete**: The instruction prompt has no substantive content that needs removal
3. **Hierarchical structure needed**: The Expected Output section needs to be structured with subsections for each documentation component

### Missing Context:
- The instruction prompt does not specify the input format (Oracle PL/SQL procedures)
- The instruction prompt does not specify the agent's role or capabilities
- The instruction prompt does not provide examples or templates
- The instruction prompt does not define quality criteria or validation rules

### Recommended Instruction Structure Enhancement:
The instruction prompt should be restructured to include:
1. **Agent Purpose**: Clear description of the documentation generation task
2. **Input Specifications**: Expected input format (Oracle PL/SQL source code)
3. **Output Specifications**: Detailed requirements for each documentation section (as remediated above)
4. **Quality Standards**: Accuracy, completeness, and formatting requirements
5. **Constraints**: What to avoid (hallucination, generic content, assumptions)
6. **Examples**: Sample output structure or templates

### Implementation Priority:
All gaps are marked as "Completed" in terms of remediation specification. The actual implementation requires:
1. Updating the instruction prompt with all specified additions
2. Organizing the Expected Output section with clear subsection headers
3. Testing the updated instruction with sample Oracle PL/SQL code
4. Validating that the agent produces output conforming to all requirements
