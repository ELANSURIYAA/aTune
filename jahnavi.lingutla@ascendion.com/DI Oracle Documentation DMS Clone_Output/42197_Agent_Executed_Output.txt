# Oracle Stored Procedure Documentation
# Procedure: bronze.usp_Load_bronze_Layer_Full

## Overview
**Procedure Name:** bronze.usp_Load_bronze_Layer_Full
**Schema:** bronze
**Type:** Orchestration Procedure
**Purpose:** Master ETL procedure that orchestrates the full refresh load of all bronze layer tables from source systems

## Description
This stored procedure serves as the main entry point for the bronze layer ETL pipeline. It executes a series of individual table load procedures in sequence, manages batch processing, tracks execution metrics, and logs comprehensive audit information for monitoring and troubleshooting purposes.

## Parameters
**Input Parameters:** None
**Output Parameters:** None

After each table load, the tables processed counter is incremented.

#### 3. Summary Statistics Calculation
The procedure queries the bronze.bz_Audit_Log table to calculate:

**Success Metrics:**
- Count of successfully loaded tables
- Total rows inserted across all successful loads

**Failure Metrics:**
- Count of failed table loads
- Total rows that failed to load

**Overall Metrics:**
- Total rows processed (sum of inserted and failed)
- Execution time in seconds

#### 4. Completion Logging
Logs comprehensive execution summary including:
- End timestamp
- Total execution time
- Tables processed, succeeded, and failed counts
- Row counts (inserted and failed)

#### 5. Master Audit Record
Inserts a master audit record into bronze.bz_Audit_Log with:
- Source: 'source_layer.*'
- Target: 'bronze.*'
- Load type: 'FULL_REFRESH_ALL_TABLES'
- All execution metrics and statistics
- Batch ID for traceability

#### 6. Transaction Commit
Commits all changes to persist audit logs and data loads.

### Error Handling

The procedure includes comprehensive exception handling:

**On Error:**
1. Captures error details:
   - Error message (SQLERRM)
   - Error number (SQLCODE)
   - Error severity, state, and line number

2. Calculates execution time up to point of failure

3. Sets overall status to 'FAILED'

4. Logs detailed error information to console output

5. Inserts error audit record into bronze.bz_Audit_Log with:
   - Error details
   - Execution metrics up to failure point
   - Failed status

6. Commits the error audit record

7. Re-throws the error using RAISE_APPLICATION_ERROR with error code -20001

## Monitoring and Troubleshooting

### Check Execution Status
```sql
-- View latest execution summary
SELECT *
FROM bronze.bz_Audit_Log
WHERE source_table = 'source_layer.*'
  AND target_table = 'bronze.*'
  AND load_type = 'FULL_REFRESH_ALL_TABLES'
ORDER BY load_timestamp DESC
FETCH FIRST 1 ROW ONLY;
```

### Check Individual Table Loads
```sql
-- View all table loads for a specific batch
SELECT 
    source_table,
    target_table,
    status,
    records_inserted,
    records_failed,
    processing_time,
    error_message
FROM bronze.bz_Audit_Log
WHERE batch_id = '<batch_id_value>'
  AND source_table != 'source_layer.*'
ORDER BY load_timestamp;
```

### Check Failed Loads
```sql
-- View failed table loads
SELECT 
    batch_id,
    source_table,
    target_table,
    error_message,
    load_timestamp
FROM bronze.bz_Audit_Log
WHERE status = 'FAILED'
ORDER BY load_timestamp DESC;
```

## Performance Considerations

1. **Sequential Processing**: Tables are loaded sequentially, not in parallel. Consider parallelization for performance improvement.

2. **Full Refresh**: All tables are fully refreshed on each execution. Consider incremental loads for large tables.

3. **Transaction Management**: Single commit at the end ensures atomicity but may hold locks longer.

4. **Audit Log Growth**: The bz_Audit_Log table will grow with each execution. Implement archival/purge strategy.

## Best Practices

1. **Scheduling**: Schedule during off-peak hours due to full refresh nature
2. **Monitoring**: Set up alerts on failed status in audit log
3. **Backup**: Ensure backup strategy is in place before execution
4. **Testing**: Test in non-production environment first
5. **Documentation**: Keep individual table load procedures documented
6. **Error Notification**: Implement email/alert notification on failures
