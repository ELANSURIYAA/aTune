Reviewer Validation Checklist

Completeness Checks

- Verify that all mandatory sections specified in the orchestration agent instructions (Sections 1–5) are present in the output.
- Confirm that every task listed in the Data SDLC Task Dependency Identifier Agent file is addressed in the mapping and gap analysis.
- Ensure that all required fields, such as agent names, mapping scores, and reasoning, are included for each mapped task.
- Check that all business domains, entities, attributes, relationships, and business rules from the input tasks are analyzed and reflected in the output.
- Validate that both available agents (from the Agent Mapper Table) and new agents (for unmapped tasks) are considered and clearly labeled as per instructions.
- Confirm that all dependencies and execution orders between agents are captured and sequenced correctly.
- Ensure that all required tables (e.g., KB/Tools/Guardrails/Models) are generated and fully populated with actionable, specific entries.
- Verify that all improvement and optimization sub-sections (efficiency gains, agent accuracy improvements, cost reduction) contain at least three lines of substantive detail each.
- Check that the run metrics section is included once, with model, API cost, and tokens used.
- Ensure that all formatting and structural requirements (minimum detail, section compliance, labeling of new agents) are followed.
- Confirm that all referenced files/content (tasks, agent mappings, requirements) are analyzed where required.
- Validate that all gap/missing item handling requirements are covered, including recommendations for new agents where needed.
- Ensure that no relevant section from the input files is ignored or omitted in the output.

Accuracy Checks

- Confirm that the output strictly aligns with the orchestration agent instructions and does not introduce unsupported assumptions or invented agent names.
- Verify that agent mappings accurately reflect the relationships and reasoning provided in the Enterprise Task Agent Mapping And Gap Analysis Orchestrator file.
- Ensure that all gap identifications and new agent recommendations are justified by the input content and not hallucinated.
- Check that dependency and technical mappings between tasks and agents are logically and technically valid.
- Validate that all business rule interpretations and terminology are consistent with the input files.
- Confirm that all findings, mappings, and recommendations are traceable to either the instructions or the input files.
- Ensure that no contradictions exist across sections, and that all business rules and requirements are interpreted correctly.
- Verify that all values, tables, and classifications accurately match the source content and that no false-positive gaps or false-negative omissions occur.
- Check that all new agents are clearly labeled as [NEW] and do not duplicate available agent functions.
- Confirm that the output does not misinterpret or partially implement any instruction or input requirement.
- Ensure that all required formatting standards (e.g., table structures, section headers) are followed precisely.
- Validate that all recommendations align with the identified findings and are supported by the input data.
- Confirm that the run metrics accurately reflect the orchestration agent’s execution and are not repeated per workflow agent.

Efficiency Checks

- Verify that duplicate analysis, redundant explanations, and repeated gap entries are avoided throughout the output.
- Ensure that the output remains concise, structured, and free of unnecessary sections or excessive verbosity.
- Check that duplicate dependency mappings, risks, recommendations, and validations are eliminated.
- Confirm that similar findings are logically grouped where appropriate, and overlapping findings are consolidated.
- Ensure that tables are not unnecessarily repeated and that large sections are summarized efficiently where allowed.
- Validate that output structure improves readability and review efficiency, with clean and optimized formatting.
- Confirm that repeated content from the input files is minimized unless explicitly required for completeness.
- Ensure that redundant classifications, tags, and checks are avoided.
- Verify that the output is token-efficient while maintaining completeness and instruction compliance.
- Check that only genuinely different workflow options are produced, and options are not generated solely to fill format requirements.
- Ensure that the run metrics section is produced only once per run and not repeated for each workflow agent.
