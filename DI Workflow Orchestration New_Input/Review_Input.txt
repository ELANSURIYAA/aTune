Reviewer Validation Checklist

Completeness Checks

- Verify that all mandatory sections specified in the agent instructions are present in the output, including both "With available agents" and "With available and new agents" sections.
- Confirm that every required field, table, and bullet-point structure described in the instructions is included in the output.
- Ensure that all agents listed in the provided Agent Mapper Table are analyzed for relevance, and that no agent outside this table is referenced as available.
- Check that all user requirements (task, role, objective, expectations) and all entries from the Agent Mapper Table are addressed in the workflow analysis.
- Validate that all identified gaps and missing capabilities are explicitly documented and that new agents are suggested only where gaps exist.
- Confirm that all required dependencies, execution orders, and agent relationships are captured and logically sequenced.
- Ensure that the KB/Tools/Guardrails/Models table is fully populated with actionable, specific entries for each column.
- Verify that the Improvements & Optimisations section covers Accuracy, Efficiency, and Cost Effectiveness with at least three actionable recommendations per dimension.
- Check that the Run Metrics section is present and includes Model Used, Tokens Used, and API Cost as specified.
- Confirm that all formatting, labeling, and sectioning requirements from the instructions are followed exactly.

Accuracy Checks

- Validate that the output strictly aligns with the provided instructions and does not introduce unsupported assumptions or hallucinated content.
- Ensure that only agents from the Agent Mapper Table are listed as available, and that new agents are clearly labeled and justified by explicit gaps.
- Check that all workflow options, agent selections, and execution orders are logically derived from the input requirements and agent mapper content.
- Confirm that all gap identifications, risk classifications, and recommendations are traceable to the input requirements or agent mapper table.
- Ensure that all technical and dependency mappings are accurate and reflect the actual relationships and requirements.
- Verify that all business rule interpretations, terminology, and classifications are consistent with the instructions and input content.
- Check that all required formatting standards, bullet-point structures, and table formats are followed precisely.
- Ensure that no false-positive gaps or false-negative omissions are present in the gap analysis and recommendations.
- Confirm that all values, tables, and classifications accurately match the source content and that no contradictions exist across sections.
- Validate that all recommendations and improvements are directly tied to the agents, tasks, or outputs involved, as required by the instructions.

Efficiency Checks

- Ensure that duplicate analysis, redundant explanations, and repeated gap entries are avoided throughout the output.
- Confirm that the output remains concise, well-structured, and free from unnecessary sections or excessive verbosity.
- Check that duplicate dependency mappings, repeated risks, and redundant recommendations are eliminated.
- Validate that tables, findings, and classifications are not unnecessarily repeated and that similar findings are logically grouped.
- Ensure that the output is token-efficient, summarizing large sections where allowed and minimizing repeated content from the input file.
- Confirm that the output structure improves readability and review efficiency, following the prescribed format without deviation.
- Check that large sections are summarized efficiently, and that output remains focused on actionable, reviewer-relevant content.
- Ensure that all validations and checks are unique, atomic, and not duplicated across sections.
- Validate that the output avoids overlapping findings and consolidates similar content where appropriate.
- Confirm that the output does not include any unnecessary filler, generic content, or sections not requested in the instructions.
