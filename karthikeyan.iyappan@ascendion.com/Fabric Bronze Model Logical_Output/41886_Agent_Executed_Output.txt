## 5. Design Decisions and Assumptions

### Key Design Decisions:

1. **Naming Convention:** All Bronze layer tables are prefixed with 'Bz_' to clearly identify them as part of the Bronze layer in the medallion architecture.

2. **Metadata Columns:** Three standard metadata columns (load_timestamp, update_timestamp, source_system) are added to all Bronze layer tables to track data lineage and processing history.

3. **Primary and Foreign Key Exclusion:** As per instructions, all primary key and foreign key fields from the source tables have been excluded from the Bronze layer logical model. The Bronze layer focuses on capturing raw data attributes without enforcing referential integrity at this stage.

4. **Data Type Preservation:** Data types from the source system are preserved in the Bronze layer to maintain data fidelity and ensure accurate representation of source data.

5. **Audit Table:** A comprehensive audit table (Bz_Audit_Log) is included to track all data processing activities, enabling monitoring, troubleshooting, and compliance reporting.

### Assumptions Made:

1. **Source System Identification:** It is assumed that the source_system column will be populated with the name or identifier of the originating system during the ETL process.

2. **Timestamp Precision:** TIMESTAMP data type is used for temporal metadata columns, assuming the platform supports this data type with sufficient precision.

3. **PII Handling:** While PII fields are identified and classified, no masking or encryption is applied at the Bronze layer, as this layer typically stores raw data. PII protection measures are expected to be implemented in Silver or Gold layers.

4. **Relationship Preservation:** Although primary and foreign key fields are excluded from the Bronze layer tables, the conceptual relationships between entities are documented to inform downstream processing in Silver and Gold layers.

5. **Data Quality:** It is assumed that basic data quality checks (e.g., null handling, data type validation) will be performed during the ingestion process, though the Bronze layer accepts data largely as-is from the source.

6. **Audit Log Population:** It is assumed that the audit table will be automatically populated by the data ingestion framework or ETL process.
