## 5. Design Decisions and Assumptions

### Key Design Decisions:

1. **Naming Convention:** All Bronze layer tables are prefixed with 'Bz_' to clearly identify them as part of the Bronze layer in the medallion architecture.

2. **Metadata Columns:** Three standard metadata columns (load_timestamp, update_timestamp, source_system) are added to all Bronze layer tables to track data lineage and processing history.

3. **Primary and Foreign Keys Exclusion:** As per instructions, all primary key and foreign key fields from the source tables have been excluded from the Bronze layer logical model. The Bronze layer focuses on capturing raw data without enforcing referential integrity.

4. **Data Type Preservation:** Data types from the source system are preserved in the Bronze layer to maintain data fidelity and avoid transformation errors.

5. **Audit Table:** A comprehensive audit table (Bz_Audit_Log) is included to track all data loading activities, enabling monitoring, troubleshooting, and compliance reporting.

### Assumptions Made:

1. **Source System Identifier:** Assumed that the source_system column will contain a standardized identifier for the originating system (e.g., "Inventory_Management_App").

2. **Timestamp Format:** Assumed that TIMESTAMP data type will capture both date and time with appropriate precision for audit and tracking purposes.

3. **Processing User:** Assumed that the processed_by field in the audit table will capture either a system service account or user identifier responsible for the data load.

4. **Status Values:** Assumed standard status values for the audit table (Success, Failed, In Progress) to track processing outcomes.

5. **Data Volume:** Assumed that the Bronze layer will handle raw data ingestion at scale, requiring appropriate partitioning and indexing strategies in the physical implementation.

6. **PII Handling:** Identified PII fields will require appropriate security measures, encryption, and access controls in the physical implementation, though these are not explicitly modeled in the logical layer.
