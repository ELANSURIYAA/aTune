Reviewer Validation Checklist

Completeness Checks

- Verify that all tasks listed in the Task Deduplication & Merge Register are mapped to an agent or explicitly addressed in the output.
- Confirm that every agent defined in the input (Requirements Parsing & Validation, Entity & Attribute Modeling, Business Rules & Compliance, Model Synthesis & Integration) is considered in the workflow options.
- Ensure that all required fields from the input tables (Task, Agent Name, Mapping Score, Reasoning, Manual Intervention) are included and analyzed in the output.
- Check that all classification types ([INPUT], [INFERRED], [RECOMMENDED]) are represented and handled according to their definitions.
- Validate that all business rules, compliance requirements, and audit trail specifications from the input are addressed in the agent mapping and workflow.
- Confirm that all dependencies, triggers, and handoff points between agents are captured and sequenced as per the input.
- Ensure that all open items and gaps listed in the agent definition cards are acknowledged and included in the workflow or gap analysis.
- Verify that all required manual interventions and human-in-the-loop steps are documented for each agent/task as specified in the input.
- Confirm that all required outputs (parsed requirements, entity list, rules catalog, conceptual data model, etc.) are present and traceable to the input content.
- Ensure that all required validations, escalation criteria, and thresholds are included as per the input recommendations.

Accuracy Checks

- Confirm that the agent mapping strictly uses only agents from the provided Agent Mapper Table and does not invent or infer agent names from requirements.
- Validate that the mapping scores and reasoning provided for each task-agent pair accurately reflect the alignment described in the input.
- Ensure that no hallucinated agents, tasks, or relationships are introduced in the output.
- Check that all dependencies and technical mappings between tasks and agents are logically consistent with the input.
- Verify that all gap identifications and unmapped tasks are justified by the input content and not based on unsupported assumptions.
- Confirm that all classifications ([INPUT], [INFERRED], [RECOMMENDED]) are applied correctly and consistently as per the input definitions.
- Ensure that all business rule interpretations, compliance requirements, and audit specifications are accurately reflected from the input.
- Validate that all terminology, task names, and agent names are used consistently and match the input exactly.
- Check that all escalation paths, thresholds, and manual intervention points are accurately represented and traceable to the input.
- Ensure that the output structure and formatting follow the requirements specified in the agent instructions and input content.

Efficiency Checks

- Confirm that duplicate analysis of tasks and agents is avoided and deduplicated as per the Task Deduplication & Merge Register.
- Ensure that redundant explanations, repeated gap entries, and unnecessary sections are not present in the output.
- Validate that the output remains concise, structured, and free of excessive verbosity.
- Check that similar findings, recommendations, or manual interventions are logically grouped and not repeated unnecessarily.
- Ensure that tables and mappings are not unnecessarily repeated and are presented in a clean, optimized format.
- Confirm that the output structure improves readability and review efficiency by following the prescribed output format.
- Validate that only essential agents and workflow steps are included, with optional or redundant agents excluded and documented as per the input.
- Ensure that large sections are summarized efficiently where allowed, and that token usage is minimized without loss of completeness.
- Confirm that all validations and checks are atomic, non-redundant, and reviewer-verifiable.
- Ensure that the output is token-efficient and avoids repeating content from the input file unless necessary for clarity or traceability.
