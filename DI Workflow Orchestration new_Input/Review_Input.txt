Reviewer Validation Checklist

Completeness Checks

- All workflow options specified in the instructions are present, including both "with available agents" and "with available and new agents" sections.
- Every agent listed in the Agent Mapper Table from the input file is evaluated for inclusion, with decisions documented for each.
- All deduplicated and merged tasks from the Consolidated Thinking Document are addressed in the agent mapping and workflow output.
- Each agent definition card's mandatory fields (purpose, tasks, triggers, business justification, technical profile, behavioural profile, guardrails, human-in-loop, explainability, restrictions, gaps) are covered in the output.
- All required analyses, such as gap identification, dependency mapping, and workflow sequencing, are included as per the instructions.
- All business rules, compliance requirements, and audit trail specifications from the input are reflected in the workflow recommendations.
- All referenced classifications ([INPUT], [INFERRED], [RECOMMENDED]) and open items in the input are addressed or flagged in the output.
- The output includes the KB/Tools/Guardrails/Models table, improvements & optimisations, and run metrics sections as mandated by the instructions.
- All required mappings between tasks and agents, as shown in the Task-Agent Mapping table, are included and justified.
- No mandatory section or required field from the instructions or input files is omitted.

Accuracy Checks

- Output aligns with the orchestration agent's instructions, including strict use of only agents from the Agent Mapper Table for the "available agents" section.
- No agent is listed as available unless it appears in the Agent Mapper Table from the input file.
- All new agents in the "with available and new agents" section are clearly labelled as [NEW] and do not duplicate available agent functions.
- Task-to-agent mappings in the output match the mappings and scores provided in the input Task-Agent Mapping table.
- All deduplication, merging, and gap identification logic from the input is accurately reflected in the workflow output.
- All dependencies and execution orders are logically correct and traceable to the requirements and agent mapping.
- No unsupported assumptions, hallucinated agents, or invented tasks are introduced in the output.
- All classifications ([INPUT], [INFERRED], [RECOMMENDED]) are used consistently and only as justified by the input.
- All business rules, compliance, and audit requirements are interpreted and represented correctly as per the input.
- No contradictions or inconsistencies exist between the workflow output and the agent definition cards or mapping tables.
- All referenced open items and gaps are explicitly addressed or flagged for validation as required.
- Output structure, terminology, and formatting match the standards and templates specified in the instructions.

Efficiency Checks

- Duplicate analysis of agents, tasks, or workflow steps is avoided in the output.
- Redundant explanations, repeated gap entries, and unnecessary sections are eliminated.
- Output remains concise, structured, and follows the prescribed format without excessive verbosity.
- Similar findings, such as overlapping gaps or recommendations, are logically grouped and not repeated.
- Tables, mappings, and lists are not unnecessarily repeated or verbose.
- Output is token-efficient, summarizing large sections where allowed and minimizing repeated content from the input files.
- All findings and recommendations are consolidated for readability and review efficiency.
- Output structure is optimized for reviewer validation, with clear separation of required sections and logical flow.
- Only required validations and checks are included; no duplicate or unnecessary validations are present.
- Output avoids unnecessary detail and focuses on actionable, instruction-aligned content.
