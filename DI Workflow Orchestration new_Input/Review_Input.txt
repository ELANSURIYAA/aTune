Accuracy Checks
- Confirm that the output aligns with the orchestration agent instructions and expected output format.
- Verify that all agent selections, workflow steps, and execution orders are justified based on the agent mapping and input requirements.
- Ensure that no agent is included in the "available agents" section unless it appears in the provided Agent Mapper Table.
- Check that new agents are only introduced in the "available and new agents" section, are clearly labeled as [NEW], and fill specific, identified gaps.
- Validate that all gap identifications and recommendations are justified by the input content and agent mapping analysis.
- Confirm that dependency mappings, technical mappings, and workflow sequences are logically correct and match the input data.
- Ensure that all classifications, tags, and statuses (e.g., [INPUT], [INFERRED], [RECOMMENDED]) are used accurately and consistently.
- Verify that all findings, recommendations, and gap identifications are traceable to the input files or agent instructions.
- Check that no unsupported assumptions, hallucinated content, or invented agent names are introduced.
- Ensure that all business rule interpretations, compliance requirements, and audit log specifications are correct and match the input.
- Validate that all relationships between entities, agents, and tasks are correctly interpreted and represented.
- Confirm that all escalation, ambiguity, and validation mechanisms are accurately reflected as per the input.
- Ensure that no contradictions or inconsistencies exist across sections, tables, or workflow options.
- Check that all required output formatting standards are followed precisely.
- Validate that no false-positive gaps or false-negative omissions occur in the mapping or recommendations.

Efficiency Checks
- Confirm that duplicate analysis, redundant explanations, and repeated gap entries are avoided in the output.
- Ensure that the output is concise, well-structured, and free from unnecessary sections or verbosity.
- Check that similar findings, recommendations, or classifications are logically grouped and not repeated.
- Validate that tables, agent lists, and workflow steps are not unnecessarily repeated or duplicated.
- Ensure that output structure and formatting improve readability and review efficiency.
- Confirm that token usage is optimized by summarizing large sections where allowed and minimizing repeated content from the input files.
- Check that only required validations and checks are included, with no redundant or duplicate entries.
- Ensure that overlapping findings or recommendations are consolidated for clarity.
- Validate that the output remains focused on instruction and input coverage, avoiding extraneous content.
- Confirm that the output supports efficient reviewer validation by maintaining atomic, reviewer-verifiable checks.
